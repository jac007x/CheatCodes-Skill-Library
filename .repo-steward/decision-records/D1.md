# Decision Record D1 — skill.yaml is the canonical config format

**Date:** 2026-03-24
**Conflict:** C1+C2 — Config file format inconsistency across 22 skills

## Standard Selected
`skill.yaml` — the only format explicitly required by GOVERNANCE.md (Gate H5).

## Rationale
GOVERNANCE.md Gate H5 states: "skill.yaml Exists — Metadata file with required fields must be present." The schema is defined in detail in GOVERNANCE.md. No equivalent requirement exists for skill.json or agent.json.

## Alternatives Considered
| Alternative | Evidence For | Evidence Against |
|-------------|-------------|-----------------|
| skill.yaml (selected) | Explicitly required in GOVERNANCE.md H5; used by 17/34 skills; schema fully documented | — |
| skill.json | Used by 5 skills; simpler format | Not mentioned as required; no schema in GOVERNANCE.md |
| agent.json | Used by designer-orchestrator; may reflect an intentional agent framework | Only 1 use; not documented as standard |

## Files Affected
All 16 skills that were missing skill.yaml — created from registry.json data + SKILL.md frontmatter.
skill.json files were preserved (not deleted) as optional extensions.

## Protected Exceptions
designer-orchestrator/agent.json preserved as documented deviation.
